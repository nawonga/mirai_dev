### Fixed
- **Jarvis sleep intent ("너 아니야"/"들어가") 응답이 거의 무음으로 들리던 버그 — Google TTS 단발 텍스트 amplitude collapse 회피.** 증상: 사용자 "너 들어가" 발화 → systemd 로그상 `intent=system.sleep`/`TTS 캐시 hit a8a16c74e8662c59.wav (len=2)`/`TTS 0.34s 완료` 모두 정상이나 스피커에서 안 들림. 진단: wav 진폭 분석 결과 `peak=1415` (16bit max 32767의 4.3%), `rms=129` — 같은 voice/rate 의 다른 응답들은 peak 25000+/rms 3000~4500 으로 약 **15~20배 큰 볼륨**. pw-play 자체는 정상 exit, 파일도 LINEAR16 24kHz 0.28s 정상 — 즉 Google Cloud TTS (ko-KR-Chirp3-HD-Leda) 가 "네." 같은 매우 짧은 입력은 self-normalization 으로 거의 무음 수준으로 합성. 해결: `src/jarvis/intent.py` system.sleep reply 를 `"네."` → `"네 알겠습니다."` (한 어절 이상으로 확장 — Google TTS 정상 볼륨 영역) + 기존 무음 캐시 wav 삭제(다음 sleep 시 자동 재합성). 라이브 재현: "향다나 → … → 너 들어가" → 정상 볼륨 "네 알겠습니다." 발화 후 wake 복귀 확인. 메모리에 `jarvis-tts-short-text-quiet` 기록 — 향후 reply 작성 시 1~2글자 짧은 응답 회피 가이드.
